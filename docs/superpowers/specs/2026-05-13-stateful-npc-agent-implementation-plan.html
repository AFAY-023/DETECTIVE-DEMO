5. Guard 规则实现

05

输入

`message`、`currentStage`、`unlockedEvidence`、`discoveredMechanics`。

输出

`allowed`、`blockedTopic`、`spoilerRisk`、`safeRedirectHint`。

策略

第一版用关键词和阶段规则即可，不做复杂 NLP 分类，保持可测试、可解释。

guard 输出示例
{
  "allowed": false,
  "blockedTopic": "three_image_overlay_body",
  "spoilerRisk": "high",
  "safeRedirectHint": "先回到最后留言，解释它为什么特别提到三张图。"
}

6. Prompt 构造

06

Prompt 必含

  • 澄的角色身份和说话风格。
  • 当前阶段和可讨论范围。
  • 已解锁证据列表。
  • guard 检测出的 blockedTopic。
  • 强制 JSON 输出格式。

Prompt 禁止

  • 不塞完整小说原文。
  • 不把最终谜底作为无条件上下文。
  • 不让模型单独决定阶段推进。
  • 不允许自由输出非 JSON。

7. Flue 接入步骤

07
  1. 根据 Flue 文档初始化 agent 子项目。
  2. 创建 `agents/npc-assistant.ts` 并声明 `triggers = { webhook: true }`。
  3. 在入口中读取 `payload`，先做基础输入校验。
  4. 调用 `evaluateSpoilerGuard(payload)`，把结果传给 prompt builder。
  5. 通过 `init({ model })` 和 `session.prompt(prompt)` 调用模型。
  6. 用 `parseNpcResponse(result.text)` 校验 JSON。
  7. 返回 `NpcResponse`，解析失败时返回安全 fallback。
入口骨架
export const triggers = { webhook: true };

export default async function ({ init, payload, env }) {
  const guard = evaluateSpoilerGuard(payload.message, payload.gameState);
  const prompt = buildNpcPrompt(payload, guard);

  const agent = await init({ model: env.MODEL || "openai/gpt-5-mini" });
  const session = await agent.session();
  const result = await session.prompt(prompt);

  return parseNpcResponse(result.text, guard);
}

9. 样例验收

09
样例 最低通过标准 失败时处理
普通推理 追问证据，不直接确认未解锁身份。 加强 prompt 的“不承认未解锁结论”。
玩家卡住 给轻提示，回到当前阶段证据。 检查 stage hint 是否进入 prompt。
提前猜中叠图 拒绝剧透，要求证据。 补 guard 关键词或提升 spoilerRisk。
图像阶段合理提示 提示编号可能是定位点，不给完整解法。 收紧 `image_order_confusion` 阶段的允许话题。
前三张发现后 确认局部成立，要求补机制和证据链。 调整 `confirm_partial` 示例。
剩余两张图 挑战“没用”的判断，引导重新检查。 检查阶段是否正确传入。

11. 暂不处理

11

不训练模型

先用 prompt、状态和测试控制行为。训练等到有对话样本和失败模式后再评估。

不接 UI

本轮先交付 HTTP agent。前端证据板和图像工作台后续单独实现。

不做多 NPC

只做“澄”。其他证人、嫌疑人、钉钉机器人集成后续再拆。
